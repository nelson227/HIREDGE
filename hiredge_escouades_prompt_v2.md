# HIREDGE -- Escouades System (Product Logic Prompt)

## Overview

The Escouades system is a collaborative feature within HIREDGE designed
to connect candidates with similar goals so they can support each other
during the job search process.

Escouades are small, private groups automatically generated by the AI
agent EDGE. Each Escouade contains **between 3 and 10 members maximum**
and is composed of candidates with **similar professional objectives but
not in direct competition**.

The system is triggered **after a user submits a job application**.

Goals: - Encourage peer support - Create a collaborative job search
experience - Allow members to prepare interviews together - Maintain
strict privacy for each candidate

------------------------------------------------------------------------

# Trigger Event

The Escouades logic begins **immediately after a user successfully
submits a job application**.

Example event:

User submits application → Application confirmed

After the confirmation, EDGE evaluates whether an Escouade opportunity
exists.

EDGE analyzes: - Job title applied to - Skills required - Seniority
level - Location (or remote) - Industry - Candidate career objectives

This information is used to identify **compatible candidates**.

------------------------------------------------------------------------

# EDGE Matching Logic

EDGE performs the following sequence:

1.  Identify Escouades already associated with the **same job domain**
2.  Filter Escouades that match:
    -   Job family (ex: Data Analyst)
    -   Experience level (Junior / Mid / Senior)
    -   Geography or remote availability
3.  Check Escouade capacity.

Conditions:

Maximum members per Escouade: **10**\
Minimum functional group: **3**

------------------------------------------------------------------------

## Case 1 --- Compatible Escouade exists with free space

EDGE adds the Escouade to the suggestion list.

------------------------------------------------------------------------

## Case 2 --- Compatible Escouade exists but is full

EDGE creates a **new Escouade**.

Example:

Escouade Data-Analyst-04 (10 members → full)

EDGE creates:

Escouade Data-Analyst-05

------------------------------------------------------------------------

## Case 3 --- No Escouade exists

EDGE automatically creates the **first Escouade for that category**.

Example:

User applies to: Cybersecurity Analyst

No Escouade exists → EDGE creates:

Escouade Cybersecurity-01

------------------------------------------------------------------------

# User Experience Flow

After the job application confirmation, a contextual message from EDGE
appears.

Important principles: - No forced redirect - No intrusive popup -
Appears in the same screen as the confirmation

Example:

Application successfully submitted.

EDGE message:

"I found candidates with a similar objective who are also applying in
this field. Joining an Escouade can help you prepare interviews
together."

Buttons:

\[ View Escouades \]\
\[ Maybe later \]

------------------------------------------------------------------------

# Escouade Cards Interface

When the user clicks **View Escouades**, a panel opens displaying
recommended Escouades.

EDGE only displays Escouades **directly related to the job applied
for**.

Each Escouade is displayed as a **UI card**.

------------------------------------------------------------------------

## Escouade Card Structure

Example:

Escouade Data Analyst -- Montréal

Members: 6 / 10

Focus: Data Analytics roles in Canada

\[ Join Escouade \]

Another example:

Escouade Business Intelligence -- Remote

Members: 5 / 10

Focus: BI / Analytics roles

\[ Join Escouade \]

Only Escouades with available slots are shown.

------------------------------------------------------------------------

# Privacy Rules Inside Escouades

Escouades respect **strict privacy principles**.

No automatic sharing of job search statistics is allowed.

Inside an Escouade, members can only see:

-   Full name of members
-   Online / Offline status

Example:

Escouade Data Analyst -- Montréal

Members (6 / 10)

Emma Dubois --- Online\
Lucas Tremblay --- Offline\
Sara Benali --- Online\
David Nguyen --- Offline\
Amélie Roy --- Offline\
Thomas Martin --- Online

------------------------------------------------------------------------

## Information That Must NEVER Be Displayed Automatically

The following data remains private:

-   number of job applications sent
-   companies applied to
-   interview invitations
-   salary negotiations
-   offers received
-   personal job search statistics

These elements are confidential and visible **only to the user
concerned**.

------------------------------------------------------------------------

## Voluntary Sharing

If a member wishes to share information, they may do so **manually
inside the Escouade chat**.

Example message:

Lucas Tremblay: "I just received an interview invitation at Ubisoft."

Sharing is always voluntary.

------------------------------------------------------------------------

# Escouade Creation Logic

If EDGE determines that:

-   No compatible Escouade exists OR
-   All Escouades are full

Then EDGE automatically creates a new Escouade.

Example:

Creating Escouade...

Escouade Data-Analyst-07

Members: 1 / 10\
Focus: Data Analyst roles in Canada

The user is invited to join this newly created Escouade.

------------------------------------------------------------------------

# Joining an Escouade

When a user presses:

Join Escouade

EDGE performs:

1.  Adds the user to the Escouade
2.  Updates member count
3.  Opens the Escouade chat space

Example message:

EDGE: "Welcome to Escouade Data Analyst-05."

------------------------------------------------------------------------

# Ignoring the Suggestion

If the user selects:

Maybe later

Then:

-   The Escouade suggestion disappears
-   The user continues normally

EDGE will not immediately propose another Escouade.

Recommended cooldown:

-   After 3 new applications OR
-   After 24 hours

------------------------------------------------------------------------

# User Preferences

Users can control Escouade suggestions in settings.

Example:

Escouade Participation

Enabled\
Suggest occasionally\
Disabled

If disabled, EDGE stops suggesting Escouades.

------------------------------------------------------------------------

# Escouade Size Limits

Maximum members per Escouade: **10**

Reasons: - Maintain real collaboration - Prevent silent oversized
groups - Keep conversations meaningful

------------------------------------------------------------------------

# Full System Flow

User applies to job ↓ Application confirmed ↓ EDGE analyzes job +
profile ↓ EDGE searches compatible Escouades ↓ Escouade exists? ↓ Yes →
Display Escouade cards No → Create new Escouade ↓ User chooses: Join
Escouade / Maybe later ↓ If joined → User enters Escouade chat

------------------------------------------------------------------------

# Core Design Principles

The Escouades system is designed around:

-   Optional participation
-   Contextual suggestions
-   Privacy-first architecture
-   AI-driven matching
-   Small collaborative groups

Escouades transform the job search from a **solitary process** into a
**collaborative experience while protecting candidate privacy**.
